    Pipeline com Motorsport Brasil, F1Mania, Grande Prêmio e Band. Classificação automática por categoria (F1, MotoGP, Stock Car, NASCAR, IndyCar, WRC). Cache de 10min + refresh de 5min no client.

  • Concluído
    UI Premium de Notícias com Filtros por Categoria

    Grid responsivo com cards dark-theme, badges de fonte, overlays de categoria, pills de filtro com contadores dinâmicos e auto-refresh sem reload.

    public/index.html · public/css/style.css
  • Concluído
    Limpeza de UI e Dados

    Remoção de referências estáticas incorretas, polimento da grade principal, versão mobile-first ajustada.

✅ FASE 2 — Sistema de Assinaturas & Cadastro

  • Concluído
    Landing Page de Planos (planos.html)

    3 tiers profissionais: Piloto Starter (R$99,90), Piloto Pro (R$149,90) e Equipe Parceira (R$499,90). Design premium dark com badges, CTA e seção de garantia.

    public/planos.html
  • Concluído
    Cadastro Plan-Aware (4 Steps)

    Fluxo inteligente: Conta → Perfil → Detalhes (Pro) ou Revisão (Starter) → Pagamento. Detecta plano via URL (?plan=starter/pro). Campos Pro extras: bio, carreira, corridas, patrocinadores, mídia kit.

    public/cadastro.html
  • Concluído
    Integração Asaas (Links de Pagamento)

    Starter: asaas.com/c/pcd93o8dzjozlmje · Pro: asaas.com/c/vz16usxyvho09ell. Após cadastro, redireciona para pagamento seguro.

  • Concluído
    Upload de foto do dispositivo + Categoria texto livre

    Foto de perfil com preview circular via FileReader. Categoria digitada livremente ao invés de dropdown fixo.

  • Concluído
    Remoção de referências a "IA" em todo o site

    Substituído por "nosso time" / "nossa redação" em cadastro, admin, dashboard-piloto e termos de uso.

    cadastro.html · admin.html · dashboard-piloto.html · termos.html
  • Concluído
    Programa de Embaixadores (Cupom de Indicação)

    Captura ?ref=ID via URL → salva no LocalStorage → preenche automaticamente no cadastro → enviado na payload de registro.

  • Concluído
    Conexão Supabase (Estrutura)

    Client Supabase configurado. API /api/db/sync faz SELECT em users e articles. Tabelas prontas para produção.

    src/lib/supabase.ts · src/app/api/db/sync/route.ts

⏳ FASE 3 — Próximas Tarefas (Prioridade Alta)

  • Próxima
    Open Graph Meta Tags (Preview no WhatsApp)

    Ao compartilhar link de notícia no WhatsApp, deve mostrar: imagem principal, título e resumo da matéria. Implementar og:title, og:description, og:image nas páginas de notícia.

  • Próxima
    Banco de Dados em Produção (Supabase)

    Migrar cadastro de localStorage (PitLane.addUser) para INSERT direto no Supabase. Criar tabela pilot_profiles com campos Pro (bio, carreira, sponsors). Validar fluxo completo de persistência.

  • Próxima
    Webhook Asaas → Supabase

    Endpoint /api/webhook/asaas que recebe confirmação de pagamento e ativa o plano do piloto no banco (coluna plan_active = true, plan_type = 'starter'|'pro').

  • Pendente
    Número de WhatsApp de Suporte

    Placeholder (5511999999999) precisa ser substituído pelo número real nos botões de contato da Equipe Parceira e Categoria.

💡 FASE 4 — Backlog (Planejado)

  • Futuro
    CMS Editorial & Automação de Matérias

    Sistema onde a redação cadastra matérias e automaticamente a página do Piloto, indexação SEO (metatags) e publicação são geradas. Integração com dados do perfil Pro.

  • Futuro
    Geração Automática de Mídia Kit PDF

    Motor de exportação para PDF com métricas, dados de pista e conquistas a partir do perfil do piloto. Para captação de patrocinadores offline.

  • Futuro
    Dashboard do Piloto com Métricas

    Painel exclusivo onde o assinante Pro visualiza views das matérias, cliques no formulário de patrocínio, e pode atualizar seus dados para novas matérias.

🏗️ Arquitetura Atual

  • Stack
    Next.js + Supabase + HTML Estático

    Frontend: HTML/CSS/JS estáticos em /public. Backend: Next.js API Routes para news RSS e DB sync.
    Banco: Supabase (PostgreSQL) — tabelas users, articles.
    Pagamento: Asaas (links diretos, webhook planejado).
    Deploy: Vercel (porta 8080 local).

  • Arquivos
    Estrutura Principal

    ● public/index.html — Homepage + News Grid
    ● public/planos.html — Landing page Planos
    ● public/cadastro.html — Cadastro 4-steps plan-aware
    ● src/app/api/news/route.ts — Agregador RSS
    ● src/app/api/db/sync/route.ts — Sync Supabase
    ● src/lib/supabase.ts — Client Supabase
